## 工艺文件结构化编辑

### 需求

类型：前后端分离的网页应用

技术栈：后端litestar，数据库使用pgsql，前端部分使用bun的nextjs，但是不使用后端功能，

一共两个页面，一个页面是模板编辑页面，用于上传模板并设计对应的模板工艺文件；
另一个页面是模板填报页面，用于生成对应的工艺文件。

模板编辑文件的本质逻辑就是上传一个pdf或者word作为背景，把背景当作图片，允许在图片的任何位置设计对应的填报点，通过设计填报的点位，填报的类型（文本、数字或者选择基础资料，多选基础资料等），制作出一个模板，注意对应的模板文件和配置需要保存在后端。

填报页面是选择设计好的模板开始填报，填报完成之后生成pdf下载。
